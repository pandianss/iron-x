# STANDARD: Educational Curriculum Modules (ODS-EDU-001)

## Module 1: Engineering Execution Systems (EES-101)
**Target:** Engineering & Computer Science
**Objective:** Teach students that code is only as good as the system that ensures it is shipped and maintained.

### Syllabus
1.  **The Myth of Heroism:** Why individual brilliance fails at scale.
2.  **Automated Enforcement:** Designing CI/CD for human behaviors.
3.  **Lab:** Build a "Commitment Engine" that locks git push access if Jira tickets are stale.
4.  **Failure Analysis:** Deconstructing a project failure using the "Systemic Failure Taxonomy."

## Module 2: Management & Operations discipline (MOD-201)
**Target:** MBA & Public Administration
**Objective:** Redefine management from "motivating people" to "designing reliable systems."

### Syllabus
1.  **The Agency Problem in Execution:** Why verbal commitments decay.
2.  **Auditability as a Virtue:** Designing transparent workflows.
3.  **The Non-Negotiable Ledger:** Governance of binding commitments.
4.  **Case Study:** The collapse of "Best Effort" cultures vs. "Enforced Standard" cultures.

## Module 3: Public Policy Implementation (PPI-301)
**Target:** Governance & Policy
**Objective:** Ensuring laws and regulations are actually enforceable.

### Syllabus
1.  **Policy vs. Physics:** Writing laws that can be algorithmically verified.
2.  **The Implementation Gap:** Measuring the distance between intent and reality.
3.  **Design Project:** Create a municipal regulation that requires *zero* human enforcement agents (fully automated compliance).

## Assessment Rubrics
**Pass/Fail Condition:**
*   Did the student rely on "trust" or "hope"? -> **FAIL**
*   Did the student expect users to "remember"? -> **FAIL**
*   Did the student build a mechanism that acts when the human sleeps? -> **PASS**
